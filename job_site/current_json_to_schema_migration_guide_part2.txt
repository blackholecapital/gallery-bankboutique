1. DEPRECATED / FOLDED / DROPPED CURRENT FIELDS
================================================================

1.1 products[].showroomDescription
    disposition: FOLD into cards[].description
    rule: When showroomDescription == oneLinePromise (true for 8 of 9
          current records), DROP showroomDescription and leave
          cards[].description unset so UI falls back to tagline.
          When distinct (Studio case), retain showroomDescription as
          cards[].description; tagline still carries oneLinePromise.

1.2 products[].orderedImages (as a flat string[])
    disposition: DEPRECATED shape. Replaced by structured
                 cards[].mainImage + cards[].media[] entries.
    post-migration: the bare array form MUST NOT appear in the new
                    dataset (R-CARD-08).

1.3 products[].orderedImagesMobile (as a flat string[])
    disposition: DEPRECATED shape. Replaced by cards[].media[]
                 entries with role="mobile".

1.4 Implicit array-index ordering on products[]
    disposition: DEPRECATED. Ordering is now driven by explicit
                 cards[].orderIndex (R-CARD-01, R-CARD-22).

1.5 No other current field is dropped. All remaining current fields
    have deterministic mapped targets (see Part 1 section 2).

================================================================
2. FALLBACK RULES APPLIED DURING MIGRATION
================================================================

2.1 Description fallback
    If showroomDescription == oneLinePromise, emit cards[].description
    as absent. Consumers fall back to cards[].tagline per schema
    description on cards[].description.

2.2 CTA kind selection (cards[].cta.kind)
    - When products[].demoUrl is present AND is the intended primary
      action: cta.kind = "external", cta.externalUrl = demoUrl.
      cta.internalPath MAY still be populated from actionPath but is
      ignored by the external branch (R-CARD-14).
    - When demoUrl is absent OR actionPath is the intended primary
      action: cta.kind = "internal", cta.internalPath = actionPath.
    - Default when ambiguous: kind = "external" (every current record
      has a demoUrl).

2.3 CTA label fallback
    cards[].cta.label default "View" (schema default) when admin has
    not supplied a per-card label.

2.4 Mobile media fallback
    When products[].orderedImagesMobile is absent (Studio, Gateway,
    Connect, PayMe Pro), emit NO role="mobile" media entries and rely
    on siteConfig.fallbacks.useDesktopImagesOnMobile = true (default,
    R-SITE-08) so desktop gallery media renders on mobile.

2.5 Pricing currency fallback
    cards[].pricing.currency inherits commerceConfig.currency and
    ultimately siteConfig.defaultCurrency "USD" (R-CARD-16, R-SITE-05).
    Migration tooling SHOULD write "USD" explicitly on each card for
    round-trip stability.

2.6 Pricing interval fallback
    cards[].pricing.interval default "month",
    cards[].pricing.intervalLabel default "mo" (R-CARD-17).

2.7 Media mime fallback
    Derive cards[].media[].mime and cards[].mainImage.mime from the
    source file extension:
        .png  -> image/png
        .jpg  -> image/jpeg
        .jpeg -> image/jpeg
        .webp -> image/webp
        .avif -> image/avif
        .gif  -> image/gif
        .svg  -> image/svg+xml
    Unknown extensions: leave null (schema permits null).

2.8 Media width/height fallback
    Probe source asset dimensions at migration; on probe failure leave
    width/height null (schema permits null).

2.9 Media alt fallback
    Default cards[].mainImage.alt and cards[].media[].alt to empty
    string "" (schema default). Admin MUST supply accessibility text
    post-migration.

2.10 Card status/active mirror
     Migrated cards SHOULD be emitted with status="active", active=true
     so existing products continue to render. R-CARD-21 mirror rule
     enforced:
        status=active    => active=true
        status=draft     => active=false
        status=inactive  => active=false

2.11 orderIndex assignment
     cards[].orderIndex = source products[] array position (0-based),
     unique across cards[] (R-CARD-22).

2.12 id derivation
     cards[].id = slug(products[].name) using Product.ts slug rule:
        id = name.toLowerCase()
                 .replace(/[^a-z0-9]+/g,'-')
                 .replace(/(^-|-$)/g,'')
     If collision, append "-" + last segment of productNumber.
     Must satisfy pattern ^[A-Za-z0-9][A-Za-z0-9._-]{1,62}[A-Za-z0-9]$.

2.13 sku derivation
     cards[].sku = products[].productNumber verbatim (e.g. "xyz.0450").

2.14 Timestamps
     cards[].createdAt = cards[].updatedAt = meta.generatedAt at
     migration run (ISO-8601 UTC, R-ROOT-02).

2.15 Site-level active flag
     siteConfig.active = true at migration so the storefront is live
     unless the operator explicitly requests offline seeding
     (R-SITE-09 / R-STATE-01).

2.16 Package limit inference
     siteConfig.packageCardLimit >= count of migrated cards with
     status="active". If unspecified by operator, set to source count
     (9) and siteConfig.packageTier = "studio" (advisory default per
     R-LIMIT-05).
