========================================================================
1. LEGEND
========================================================================

1.1 target_domain values: siteConfig | cards | commerceConfig | meta
1.2 disposition values:
    - MAP         = current field migrates directly into target domain
    - TRANSFORM   = current field migrates with shape change
    - SPLIT       = current field splits into multiple target fields
    - FOLD        = current field folds into an existing target field
    - DROP_CANDIDATE = redundant in current data, likely removed
    - NEW         = no current field; must be supplied by admin/default
1.3 notation: current.path  ->  target_domain.path

========================================================================
2. CURRENT -> TARGET MAP (PER CURRENT FIELD)
========================================================================

2.1 products[]
    current: products (array<object>)
    target: cards (array<object>)
    disposition: TRANSFORM (rename products -> cards; enforce card limit
                 from siteConfig.packageCardLimit)

2.2 products[].name
    current_type: string
    target: cards[].title
    disposition: MAP

2.3 products[].productNumber
    current_type: string ("xyz.04xx")
    target: cards[].sku (primary) and cards[].id (derived if no UUID
             assigned yet)
    disposition: SPLIT
    migration_note: treat productNumber as sku; generate a separate
                    stable cards[].id (UUID or slug) at migration.

2.4 products[].oneLinePromise
    current_type: string
    target: cards[].tagline
    disposition: MAP

2.5 products[].showroomDescription
    current_type: string
    target: cards[].description
    disposition: FOLD
    migration_note: on 8 of 9 records this equals oneLinePromise;
                    when equal, fold into cards[].description using
                    oneLinePromise value. When distinct (Studio),
                    keep showroomDescription as cards[].description
                    and keep oneLinePromise as cards[].tagline.

2.6 products[].buyerOutcome
    current_type: string
    target: cards[].outcome
    disposition: MAP

2.7 products[].actionPath
    current_type: string ("/profile/<slug>")
    target: cards[].cta.internalPath
    disposition: MAP
    migration_note: pair with ctaKind = "internal" when demoUrl absent
                    or not preferred.

2.8 products[].demoUrl
    current_type: string (absolute URL)
    target: cards[].cta.externalUrl
    disposition: MAP
    migration_note: pair with ctaKind = "external" when demoUrl is
                    the primary action.

2.9 products[].orderedImages
    current_type: array<string> (relative paths)
    target: cards[].media[] with role="gallery" AND cards[].mainImage
            (explicit, derived from index 0 at migration)
    disposition: SPLIT
    migration_note:
      - index 0 -> cards[].mainImage = { url|r2Key, alt, role:"main" }
      - indexes 1..n -> cards[].media[] with role:"gallery",
        orderIndex preserved from array position.
      - All strings must be promoted to structured media objects
        carrying url or r2Key, alt, mime, width, height placeholders.

2.10 products[].orderedImagesMobile
    current_type: array<string>, present on 5 of 9
    target: cards[].media[] with role="mobile"
    disposition: MAP (conditional)
    migration_note:
      - If present, append as media entries with role="mobile" and
        preserved orderIndex.
      - If absent, leave mobile set empty and rely on siteConfig
        fallback behavior (desktop media used on mobile).

2.11 products[].bullets
    current_type: array<string>
    target: cards[].bullets
    disposition: MAP

2.12 products[].pricing
    current_type: object
    target: cards[].pricing
    disposition: TRANSFORM
    migration_note: expand with currency (default USD from
                    commerceConfig.currency) and interval label.

2.12.1 products[].pricing.setup
    target: cards[].pricing.setup (number)
    disposition: MAP

2.12.2 products[].pricing.monthly
    target: cards[].pricing.monthly (number)
    disposition: MAP

4. DROP / FOLD CANDIDATES

4.1 products[].showroomDescription
    disposition: FOLD into cards[].description (see 2.5).

========================================================================
5. PLACEHOLDERS REQUIRING MANUAL ADMIN COMPLETION AT MIGRATION
========================================================================

5.1 All siteConfig fields (Section 3.1) — no current source.
5.2 All commerceConfig fields (Section 3.3) — no current source.
5.3 All meta fields (Section 3.4) — no current source; partially
    auto-populated by migration tooling (schemaVersion, source,
    generatedAt).
5.4 cards[].media[] metadata (alt, width, height, mime, r2Key) —
    partial auto-derive (mime from extension, width/height via probe),
    remainder requires admin entry.
5.5 cards[].thumbnail — no current source; admin-supplied or derived
    by resizing cards[].mainImage at migration tooling stage.
5.6 cards[].cta.kind and cards[].cta.label — no current source; admin
    supplies label, kind chosen per record based on which of
    actionPath or demoUrl is the primary action.
5.7 cards[].pricing.currency and cards[].pricing.interval — not
    declared in source; defaults from commerceConfig.currency and
    "monthly" respectively.
5.8 Mobile media for Studio, Gateway, Connect, PayMe Pro — absent in
    source (see 2.10); siteConfig fallback or admin upload required.
