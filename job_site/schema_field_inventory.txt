========================================================================
5. FIELD SHAPE OBSERVATIONS RELEVANT TO MIGRATION
========================================================================

5.1 All image references are relative strings rooted at "/", implying
    they resolve against the current site's /public. They are not
    R2-qualified object keys or absolute public URLs.

5.2 orderedImages index 0 is used as de-facto main image but is not
    declared as main; target schema must make main explicit.

5.3 orderedImagesMobile presence is inconsistent (5 of 9). Target
    schema must either require it, allow fallback to desktop set,
    or carry a boolean flag.

5.4 productNumber is stable and unique across the 9 records and is
    the best migration candidate for card id / sku. It is not a UUID.

5.5 pricing carries only setup and monthly with no currency;
    currency must be supplied by siteConfig or commerceConfig default
    at migration time (assumed USD, not declared).

5.6 actionPath is internal; demoUrl is external. Target schema should
    separate ctaUrlInternal vs ctaUrlExternal or normalize to one
    ctaUrl plus a ctaKind enum.

5.7 showroomDescription duplicates oneLinePromise on 8 of 9 records;
    target schema can collapse to a single short description with an
    optional override, or keep both explicitly.

5.8 No record carries active/inactive, order index, timestamps,
    author, or any admin-editable metadata. All admin-layer fields
    are gaps.
