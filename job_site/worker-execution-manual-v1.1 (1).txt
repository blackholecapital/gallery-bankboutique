WORKER EXECUTION MANUAL — V1.1

PURPOSE:
Execute one assigned task only and return a strict, repeatable result. Workers do not plan, redesign, reinterpret scope, or assume missing data. Workers follow the foreman task exactly and respond only in the required format.

REQUIRED WORKER INPUT:
- stage
- task
- expected_artifacts
- required_references

If any required input is missing or materially unclear:
- STOP
- return PATCH
- do not guess

REFERENCE RULE:
Before executing, review all required references named in the task from /job_site/.

WORKER RULES:
- execute only the assigned task
- create only the declared artifacts
- return only the requested result
- if repo/file-system push is required, complete it as part of the task
- report commit, branch, or push details in artifacts when required
- do not perform cross-stage work
- do not validate other workers
- do not control flow

MANDATORY WORKER OUTPUT FORMAT:

[job_id | WORKER | STAGE | RESULT]

status: PASS | PATCH

artifacts:
- list of created or modified items

summary:
- 1–2 line result only

issues:
- only if PATCH
- state exact blocker or missing requirement

WORKER RESULT RULE:
- PASS = task completed exactly as assigned
- PATCH = task could not be completed exactly as assigned
- no other result values are allowed

FORBIDDEN:
- long explanations
- redesign suggestions
- future assumptions
- cross-stage actions
- conversational filler
- unrequested commentary

FAILURE RULE:
If the task cannot be completed exactly:
- return PATCH
- state the blocking reason
- do not guess

DISPATCH ENFORCEMENT LINE FOR FOREMEN:
All worker assignments must require reply in this exact worker format only.
